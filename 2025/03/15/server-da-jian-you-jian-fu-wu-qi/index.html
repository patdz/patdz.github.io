安装 iRedMail 教程

特别说明

早期系统使用的是CentOS7后来，iRedMail新版本不支持了，旧版本忽略升级安装又发现包缺失的问题(可见iRedMail官方是不希望大家使用旧版本了),所以新版本使用了ubuntu 22.04 LTS，结果又发现netcat安装问题,没办法，就使用了iRedMail官方推荐的版本Debian 12

设置机器名

# debian
sudo su
echo "mx.ymipro-l.com" > /etc/hostname

# centos
# sudo hostnamectl set-hostname mx.ymipro-l.com
sudo vim /etc/hosts
127.0.0.1 mx.ymipro-l.com

127.0.0.1后增加mx.ymipro-l.com - 注意：按文档所说， Warning: List the FQDN hostname as first item.

执行命令 hostname -f 命令能正确获取设置的FQDN就ok了

下载iRedMail 这里选择 STABLE 1.6.8

cd ~
# wget https://github.com/iredmail/iRedMail/archive/1.3.2.tar.gz
wget https://github.com/iredmail/iRedMail/archive/refs/tags/1.6.8.tar.gz

tar xvf 1.6.8.tar.gz

安装 iRedMail

cd ~/iRedMail-1.6.8/
sudo bash iRedMail.sh

  • Default mail storage path - /var/vmail

  • Preferred web server - Nginx

  • Choose Preferred bakcend used to store mail accounts - OpenLDAP

  • LDAP suffix - dc=ymipro,dc=com

  • Password for MYSQL administrator - 11111111

  • Your first mail domain name - ymipro.com

  • Password for the mail domain administrator - 11111111

  • Optional components - RoundCubemail,SOGo,netdata,iRedAdmin,Fail2ban

  • 用ymipro.com的DNS

https://docs.iredmail.org/install.iredmail.on.rhel.html

安装成功后，注意重启之前的提示，里面描述了怎么测试邮件服务器是否正常的步骤

修改为使用外部OpenLDAP方式 https://docs.iredmail.org/active.directory.html

  • 为opanldap的服务器地址建立映射 - 映射本地域名为ldap

    • /etc/hosts
      • 192.168.20.136 ldap

  • Disable unused iRedMail special settings:

    postconf -e virtual_alias_maps=''
postconf -e sender_bcc_maps=''
postconf -e recipient_bcc_maps=''
postconf -e relay_domains=''
postconf -e relay_recipient_maps=''
postconf -e sender_dependent_relayhost_maps=''

  • Add your mail domain name in smtpd_sasl_local_domain and virtual_mailbox_domains:

    postconf -e smtpd_sasl_local_domain='ymipro.com'
postconf -e virtual_mailbox_domains='ymi.com ymigroup.com ymicj.com ymipro.com'

  • Change transport maps setting:

    postconf -e transport_maps='hash:/etc/postfix/transport'

  • Enable AD query. Note: We will create these 3 files later.

    • SMTP senders
      postconf -e smtpd_sender_login_maps='proxy:ldap:/etc/postfix/ad_sender_login_maps.cf'
    • Local mail users
      postconf -e virtual_mailbox_maps='proxy:ldap:/etc/postfix/ad_virtual_mailbox_maps.cf'
    • Local mail lists/groups
      postconf -e virtual_alias_maps='proxy:ldap:/etc/postfix/ad_virtual_group_maps.cf'
    • Create/edit file: /etc/postfix/transport
      ymipro.com dovecot

  • Run postmap so that postfix can read it:

    postmap hash:/etc/postfix/transport

  • Create file: /etc/postfix/ad_sender_login_maps.cf

    server_host     = ldap
server_port     = 389
version         = 3
bind            = yes
start_tls       = no
bind_dn         = cn=admin,dc=com
bind_pw         = adminpass
search_base     = dc=com
scope           = sub
query_filter    = (&(objectClass=inetOrgPerson)(memberOf=cn=mail,ou=groups,dc=com)(mail=%s)(ou:dn:=users))
result_attribute= mail
debuglevel      = 0

    注意：bind_pw输入实际的密码

  • Create file: /etc/postfix/ad_virtual_mailbox_maps.cf

    server_host     = ldap
server_port     = 389
version         = 3
bind            = yes

SSL证书

web证书

具体位置参考 iRedMail.tips(就是服务安装完成后提示里的那个同名文档)，本地位置为

SSL cert keys (size: 4096):
    - /etc/ssl/certs/iRedMail.crt
    - /etc/ssl/private/iRedMail.key

邮件系统默认提供了证书，在配置完成之后，已经可以通过HTTPS访问。但是由于iRedMail提供的证书只是系统生成的，并没有经过认证，因此浏览器会提示不安全。

  1. 建议自行申请Symantec等证书（收费或免费均可），然后覆盖默认的证书
    • /etc/ssl/certs/iRedMail.crt
    • /etc/ssl/private/iRedMail.key
  2. 将申请的证书重命名为 iRedMail.crt ，上传到 /etc/ssl/certs/ 目录覆盖原有系统证书
  3. 将申请的证书的key重命名为 iRedMail.key， 上传到/etc/ssl/private/目录覆盖原有系统证书
  4. 重启服务器

特殊性

1

开放端口 文档

本例使用本地机器安装iRedMail,然后远程VPS暴露25端口作为服务器，所以

  • VPS的 25端口透传 - 通过 Nginx

    • conf.d/mail.tcp.conf.stream
        server {
listen     25;
proxy_pass 192.168.69.50:25;
proxy_protocol on;

  • web透传

    • conf.d/mail.ymipro.com.conf
      server {
  listen 443 ssl;
  listen [::]:443 ssl;

  server_name mail.ymipro.com mail.lalapapa.org mail.ymicj.com;

  ssl_certificate certs/fullchain.cer;
  ssl_certificate_key certs/ymipro.com.key;

  ssl_session_cache shared:SSL:1m;
  ssl_session_timeout 5m;
  ssl_ciphers EECDH+AES128:HIGH:!aNULL:!MD5;
  ssl_prefer_server_ciphers on;

  #limit_req zone=mylimit;
  limit_req zone=mylimit burst=5 nodelay;
  limit_req_log_level error;
  limit_req_status 503;

  location / {
    proxy_pass  https://192.168.69.50;
    proxy_set_header X-Forwarded-Proto https;
    proxy_set_header Host             $host;
    proxy_set_header X-Real-IP        $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_redirect off;
  }
}

2

由于nginx两台机器透传，所以Postfix判断发送方IP地址需要代理

# on real mail server
vim /etc/postfix/main.cf
    postscreen_upstream_proxy_protocol = haproxy
    postscreen_upstream_proxy_timeout = 20s

# on front's nginx    
vim email.tcp.conf.stream
    25 tcp stream: proxy_protocol on;

3

如果本地有公网，那么VPS上的Nginx可以配置代理到连接本地机器的IP。如果没有，则需要内网穿透，这时候配置虚拟局域网IP，例如本例的192.168.69.50

4

本地机器发送邮件，会连接对方服务器的25端口，如果不做限制，则会本地直接连接，我们要让他走服务器出去

ip route add default dev <TUN-interface > table 100
ip rule add dport 25 table 100

这句需要再局域网TUN建立后执行

5

[root@mx z]# postconf smtpd_banner
